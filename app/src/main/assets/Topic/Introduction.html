Data Structure:
It is a way of organising and storing data in a computer,or In other words
It is a memory representation of data


Primitive Data Structures:They are the basic data structures that directly operate upon the machine instructions. They have different representations on different computers. e.g Integers, floating point numbers, character constants, pointers etc.

Non-primitive data structures:They are derived from primitive data structures. They emphasize on grouping same or different data items with relationship between each data item.e.g Arrays, lists,Stack etc.

Linear data structure: A data structure is said to be linear if the elements form a sequence i.e., while traversing sequentially, we can reach only one element directly from another. For example : Array, Linked list, Queue etc.

Non linear data structure : A data structure is said to be non-linear if the elements do not form a sequence i.e each item or element may be connected with two or more other items or form a hierarchical classification.For example : Trees and Graphs etc.

Homogenous data structure: All Elements of a data structure belongs to the same data type. Ex : Array of integers contains only elements of 'int' data type, Array of characters contains only elements of 'char' data type.

Non-Homogenous data structure: Elements belongs to the different datatypes. Ex: Struture of student records contains elements of 'int' datatype for student_id and elements of 'char' datatype for Name and Address e.t.c

Dynamic data structure :data structure that grows or shrinks in size as per the program requirement is called dynamic data structure.Memory is allocated dynamically at run-time using Memory Allocating Functions e.g Linked list.

Static data structure :They have fixed size and memory has to be allocated in advance e.g Arrays

Abstract Datatype:
The data type which does not specify an implementation is an abstract data type. It is a theoretical construct that consists of data as well as the operations to be performed on the data while hiding the implementation i.e We have only concern about data and the operations performed by the dataStrucure not how it is actually implemented.e.g a stack (which is an abstract type) can be implemented as an array, or as a linked list,but we are only concerned about the operations and the data stored by the stack.
