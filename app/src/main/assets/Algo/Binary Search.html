Binary search
Binary Search follows Divide and Conquer Strategy.The idea behind binary search is that each time we make a comparison, we eliminate half of the list, until we either find the search item or deitemine that the item is not in the list. We do this by looking at the middle item in the list, and deitemining if our search item is higher or lower than the middle item. If it's lower, we eliminate the upper half of the list and repeat our search starting at the point halfway between the first item and the middle item. If it's higher, we eliminate the lower half of the list and repeat our search starting at the point halfway between the middle item and the last item.
Note:Binary Search works only, if the list is sorted.

Algorithm for binary search: 
set first = 1, last = N, mid = N/2

while (item not found and first is less than last) {

compare search item to item at mid

if match

save index

break

else if search item is less than item at mid

set last = mid-1

else

set first = mid+1

set mid = (first+last)/2

}

return index of matching item, or -1 if not found

Performance of Binary Search

Best case:The best case for binary search occurs when we find the search term on the first try. In this case, the search term would be in the middle of the list. As with linear search, the best case complexity for binary search is O(1).

Worst case:The worst case for binary search occurs when the search term is not in the list, or when the search term is one item away from the middle of the list, or when the search term is the first or last item in the list.worst case complexity for binary search is O(logN).

Average case:The average case occurs when the search term is anywhere else in the list. The number of comparisons is roughly the same as for the worst case, so case average complexity for binary search is O(logN).
