Linear search:
The simplest search algorithm is linear search.In linear search, we look at each item in the list in turn, quitting once we find an item that matches the search item or once we've reached the end of the list. Our "return value" is the index at which the search item was found, or some indicator that the search item was not found in the list.It is also known as sequential search technique.

Algorithm for linear search:
for (each item in list) {

compare search item to current item

if match

save index of matching item

break

}

return index of matching item, or -1 if item not found

Performance of linear search

Best case: The best case occurs when the search item is in the first slot in the array. The number of comparisons in this case is 1.Thus the best case complexity is:O(1)

Worst case: The worst case occurs when the search item is in the last slot in the array, or is not in the array. The number of comparisons in this case is equal to the size of the array. If our array has 'N' elements.Then the worst case complexity is:O(N)

Average case: On average, the search item will be somewhere in the middle of the array. The number of comparisons in this case is approximately N/2 but the number of comparisons are also proportional to the number of items in the array.Thus the average case complexity is:O(N)
